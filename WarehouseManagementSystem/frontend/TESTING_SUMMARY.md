# 测试总结

## 测试方法概览

### 1. 手动测试（推荐先做）
**时间：** 5-30分钟
**工具：** 浏览器
**步骤：**
1. 启动开发服务器：`npm run dev`
2. 打开浏览器访问 `http://localhost:5173`
3. 按照 `QUICK_TEST.md` 或 `TESTING_GUIDE.md` 进行测试

**优点：**
- 快速发现UI问题
- 真实用户体验
- 无需编写测试代码

**缺点：**
- 容易遗漏边界情况
- 不可重复
- 耗时

### 2. 单元测试
**时间：** 5-10分钟
**工具：** Vitest
**命令：** `npm run test`

**优点：**
- 快速反馈
- 可重复执行
- 易于维护

**缺点：**
- 需要编写测试代码
- 不能测试UI交互

### 3. 集成测试
**时间：** 10-20分钟
**工具：** Vitest + Vue Test Utils
**步骤：**
1. 编写组件测试
2. 运行测试：`npm run test`

**优点：**
- 测试组件交互
- 测试数据流

**缺点：**
- 需要编写测试代码
- 执行时间较长

## 推荐测试流程

### 第一阶段：快速验证（5分钟）
```bash
# 1. 启动开发服务器
npm run dev

# 2. 打开浏览器进行快速测试
# 参考 QUICK_TEST.md
```

### 第二阶段：完整功能测试（20分钟）
```bash
# 1. 继续使用开发服务器
# 2. 按照 TESTING_GUIDE.md 进行完整测试
# 3. 检查浏览器控制台是否有错误
```

### 第三阶段：单元测试（5分钟）
```bash
# 1. 运行单元测试
npm run test

# 2. 查看测试结果
# 3. 生成覆盖率报告
npm run test:coverage
```

### 第四阶段：代码质量检查（2分钟）
```bash
# 1. 运行ESLint检查
npm run lint

# 2. 格式化代码
npm run format
```

## 常见问题

### Q: 如何快速测试？
A: 使用 `QUICK_TEST.md` 中的5分钟快速测试流程

### Q: 如何调试问题？
A: 
1. 打开浏览器开发者工具（F12）
2. 查看Console标签中的错误
3. 查看Network标签中的API请求
4. 设置断点进行调试

### Q: 如何运行单元测试？
A: 运行 `npm run test`

### Q: 如何生成测试覆盖率报告？
A: 运行 `npm run test:coverage`

### Q: 如何修复ESLint错误？
A: 运行 `npm run lint`
