# 前端项目测试指南

## 1. 启动开发服务器

首先，确保后端API服务已启动（通常在 `http://localhost:5000`）。

### 安装依赖
```bash
cd frontend
npm install
```

### 启动开发服务器
```bash
npm run dev
```

应用将在 `http://localhost:5173` 启动。

## 2. 手动测试流程

### 2.1 测试登录功能

1. 打开浏览器访问 `http://localhost:5173`
2. 应该自动重定向到 `/login` 页面
3. 输入用户名和密码（根据后端配置）
4. 点击"登录"按钮
5. 验证：
   - ✅ 登录成功后重定向到首页
   - ✅ 用户信息显示在右上角
   - ✅ 令牌保存到localStorage

### 2.2 测试导航功能

1. 在左侧菜单中点击不同的菜单项
2. 验证：
   - ✅ 仪表板页面正常显示
   - ✅ 储位管理页面正常显示
   - ✅ 任务管理页面正常显示

### 2.3 测试储位管理

#### 列表和搜索
1. 进入"储位管理"页面
2. 验证：
   - ✅ 储位列表正常加载
   - ✅ 分页功能正常
   - ✅ 搜索功能正常

#### 详情弹框
1. 点击任意储位行
2. 验证：
   - ✅ 弹框打开显示储位详情
   - ✅ 显示所有储位信息（名称、分组、物料代码等）

#### 清空物料
1. 在详情弹框中点击"清空物料"按钮
2. 验证：
   - ✅ 确认对话框出现
   - ✅ 操作成功后显示成功提示
   - ✅ 列表自动刷新

#### 锁定/解锁
1. 在详情弹框中点击"锁定"或"解锁"按钮
2. 验证：
   - ✅ 操作成功后显示成功提示
   - ✅ 列表自动刷新

#### 批量操作
1. 勾选多个储位的复选框
2. 验证：
   - ✅ 批量操作工具栏出现
   - ✅ 显示已选择的数量
3. 点击"批量清空物料"
4. 验证：
   - ✅ 确认对话框出现
   - ✅ 操作成功后显示成功提示
   - ✅ 列表自动刷新

#### 创建/编辑储位
1. 点击"新增储位"按钮
2. 验证：
   - ✅ 跳转到创建页面
   - ✅ 表单正常显示
3. 填写表单并提交
4. 验证：
   - ✅ 创建成功后返回列表页面
   - ✅ 新储位出现在列表中

### 2.4 测试任务管理

#### 列表和过滤
1. 进入"任务管理"页面
2. 验证：
   - ✅ 任务列表正常加载
   - ✅ 分页功能正常
3. 选择日期范围并点击"筛选"
4. 验证：
   - ✅ 列表按日期范围过滤

#### 任务详情
1. 点击任意任务行
2. 验证：
   - ✅ 跳转到任务详情页面
   - ✅ 显示所有任务信息

#### 创建任务
1. 点击"新增任务"按钮
2. 验证：
   - ✅ 跳转到创建页面
   - ✅ 位置选择器正常显示
3. 选择源位置和目标位置
4. 验证：
   - ✅ 自动检查重复任务
   - ✅ 如果重复显示警告
5. 填写其他信息并提交
6. 验证：
   - ✅ 创建成功后返回列表页面
   - ✅ 新任务出现在列表中

#### 取消任务
1. 在任务详情页面点击"取消任务"
2. 验证：
   - ✅ 确认对话框出现
   - ✅ 操作成功后显示成功提示
   - ✅ 任务状态更新为"已取消"

### 2.5 测试登出功能

1. 点击右上角用户菜单
2. 点击"退出登录"
3. 验证：
   - ✅ 重定向到登录页面
   - ✅ localStorage中的令牌被清除
   - ✅ 无法直接访问受保护的页面

## 3. 浏览器开发者工具检查

### 3.1 检查网络请求
1. 打开浏览器开发者工具（F12）
2. 进入"Network"标签
3. 执行各种操作
4. 验证：
   - ✅ 所有API请求都返回200或正确的状态码
   - ✅ 请求头中包含Authorization令牌
   - ✅ 响应数据格式正确

### 3.2 检查控制台错误
1. 打开浏览器开发者工具（F12）
2. 进入"Console"标签
3. 执行各种操作
4. 验证：
   - ✅ 没有JavaScript错误
   - ✅ 没有警告信息

### 3.3 检查本地存储
1. 打开浏览器开发者工具（F12）
2. 进入"Application"标签
3. 查看"Local Storage"
4. 验证：
   - ✅ `auth_token` 已保存
   - ✅ `auth_user` 已保存

## 5. 常见问题排查

### 问题1：无法连接到后端API
**症状：** 登录失败，显示网络错误
**解决方案：**
1. 检查后端API是否运行在 `http://localhost:5000`
2. 检查 `.env` 文件中的 `VITE_API_URL` 配置
3. 检查浏览器控制台的网络请求

### 问题2：登录后无法访问页面
**症状：** 登录成功但页面显示空白或重定向到登录页
**解决方案：**
1. 检查浏览器控制台是否有错误
2. 检查localStorage中是否保存了令牌
3. 检查路由守卫是否正确配置

### 问题3：API请求返回401错误
**症状：** 操作失败，显示"无效的令牌"
**解决方案：**
1. 检查令牌是否过期
2. 重新登录获取新令牌
3. 检查后端JWT配置

### 问题4：表格数据不显示
**症状：** 列表页面打开但没有数据
**解决方案：**
1. 检查浏览器控制台是否有错误
2. 检查API是否返回正确的数据格式
3. 检查后端数据库是否有数据

## 6. 性能测试

### 6.1 检查页面加载时间
1. 打开浏览器开发者工具（F12）
2. 进入"Performance"标签
3. 点击"Record"按钮
4. 执行操作
5. 点击"Stop"按钮
6. 查看性能报告

### 6.2 检查内存使用
1. 打开浏览器开发者工具（F12）
2. 进入"Memory"标签
3. 点击"Take heap snapshot"
4. 执行操作
5. 再次点击"Take heap snapshot"
6. 比较两个快照
