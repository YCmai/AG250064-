# 仓库管理系统：新增数据库表开发指南

本指南详细说明了在此系统中新增一张数据库表，并实现其基础 CRUD（增删改查）逻辑的标准步骤。

---

## 第一步：创建数据库表 (SQL Server)

按现有规范，所有的业务表名应当以 **`RCS_`** 作为前缀。

```sql
CREATE TABLE [dbo].[RCS_MyNewTask] (
    [Id]            INT            IDENTITY (1, 1) NOT NULL,
    [Code]          NVARCHAR (50)  NULL,
    [Status]        INT            NOT NULL DEFAULT 0,
    [CreatedTime]   DATETIME       NOT NULL DEFAULT GETDATE(),
    -- 业务所需其它字段
    CONSTRAINT [PK_RCS_MyNewTask] PRIMARY KEY CLUSTERED ([Id] ASC)
);
```

---

## 第二步：创建实体模型 (Model)

在 `Models/Ndc/` 文件夹下创建 C# 文件。

1.  **文件路径**：`Models/Ndc/MyNewTask.cs`
2.  **继承基类**：建议继承 `NdcEntityBase<int>`，它会自动包含 `Id` 字段。
3.  **命名空间**：`WarehouseManagementSystem.Models.Ndc;`

```csharp
using WarehouseManagementSystem.Models.Ndc;

namespace WarehouseManagementSystem.Models.Ndc;

public class MyNewTask : NdcEntityBase<int>
{
    public string Code { get; set; }
    public int Status { get; set; }
    public DateTime CreatedTime { get; set; } = DateTime.Now;
}
```

## 第三步：注册表名映射 (Mapping)

系统中使用了 Dapper 泛型仓储。你需要告诉系统这个模型对应数据库里的哪张表。

1.  **打开文件**：`Infrastructure/Ndc/DapperEntityRepository.cs`
2.  **定位方法**：`GetTableName(Type type)`
3.  **增加 Case**：

```csharp
// 在 GetTableName 中按字母顺序或随意位置增加：

```

## 第四步：创建服务层 (Service)

通常在 `Services/Rcs/IRcsTaskDataService.cs` 中增加。

### 1. 定义接口 (Interface)
```csharp
public interface IRcsMyNewTaskService
{
    Task<List<MyNewTask>> GetAllTasksAsync();
    Task<MyNewTask?> GetByIdAsync(int id);
    Task InsertAsync(MyNewTask entity);
}
```

### 2. 实现类 (Implementation)
让实现类继承 `RcsDapperDataServiceBase`，可以复用基类简单的增删改查：

```csharp
public sealed class RcsMyNewTaskService : RcsDapperDataServiceBase, IRcsMyNewTaskService
{
    public RcsMyNewTaskService(IDatabaseService db) : base(db) { }

    public Task<List<MyNewTask>> GetAllTasksAsync()
    {
        return LoadAllAsync<MyNewTask>("RCS_MyNewTask");
    }

    public Task<MyNewTask?> GetByIdAsync(int id)
    {
        return FindFirstAsync<MyNewTask>("RCS_MyNewTask", x => x.Id == id);
    }

    public Task InsertAsync(MyNewTask entity)
    {
        return InsertEntityAsync("RCS_MyNewTask", "Id", entity);
    }
}
```

---

## 第五步：注册服务 (Dependency Injection)

如果不注册，程序在运行到构造函数注入时会崩溃报错。

1.  **打开文件**：`Program.cs`
2.  **注册服务**：

```csharp
// 在其他 AddScoped 下方增加一行
builder.Services.AddScoped<IRcsMyNewTaskService, RcsMyNewTaskService>();
```

## 💡 开发提示 (Pro Tips)

-   **Namespace 修复**：如果你发现报错说找不到类，请在文件顶部添加：
    -   `using WarehouseManagementSystem.Models.Ndc;`
    -   `using WarehouseManagementSystem.Services.Rcs;`
-   **数据库事务**：如果你在执行过程中需要包含多个数据库操作（比如插入任务的同时锁定库位），请参考 `ILocationService.cs` 中的 `transaction` 使用方式：
    ```csharp
    using var connection = _db.CreateConnection();
    connection.Open();
    using var transaction = connection.BeginTransaction();
    // 执行 Dapper 语句并传入 transaction 即可
    ```
